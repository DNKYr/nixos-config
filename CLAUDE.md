# nixos-config

> DNKYr's NixOS flake managing two hosts declaratively: `aether` (AMD laptop, full desktop) and `ikuyo` (Hetzner VPS, headless server). System config + home-manager + agenix-managed secrets, all assembled via flake-parts and import-tree.

---

## Quick Reference

| Task | Command |
|------|---------|
| Rebuild & switch (current host) | `sudo nixos-rebuild switch --flake .` (from `/home/dnkyr/nixos-config`) |
| Rebuild a specific host | `sudo nixos-rebuild switch --flake .#aether` (or `.#ikuyo`) |
| Test without switching | `sudo nixos-rebuild test --flake .` |
| Build without activating | `sudo nixos-rebuild build --flake .` |
| Update flake inputs | `nix flake update` |
| Garbage collect | `sudo nix-collect-garbage -d` (automated weekly via `modules/features/optimize.nix`) |
| Edit an agenix secret | `cd secrets && agenix -e <name>.age` |
| Re-key secrets after key change | `cd secrets && agenix -r` |
| Provision a fresh ikuyo VPS | `nix run github:nix-community/nixos-anywhere -- --flake .#ikuyo --target-host root@<ip>` |

**Non-negotiables:**
- Never edit `modules/hosts/<host>/hardware.nix` — auto-generated by `nixos-generate-config` (aether) or by nixos-anywhere (ikuyo)
- Always declare `inputs` in home-manager module function args when the module needs a flake input (e.g., `nix-gaming`, `zen-browser`, `noctalia`, `emtodo`) — forgetting this causes an eval error
- Feature modules under `modules/features/` MUST follow the flake-parts pattern (`{ ... }: { flake.nixosModules.<name> = { ... }: { ... }; }`) and be imported by name in the host's `configuration.nix` — `import-tree` only registers them, it does not auto-activate them
- Never bump `home.stateVersion` or `system.stateVersion` without reading the NixOS release notes — these are one-way migrations that can break stateful data
- Raw configs (`configs/niri/`, `configs/nvim/`) are symlinked by home-manager; edit the source in `configs/`, not the deployed path in `~/.config/`
- The hibernate setup (`modules/hosts/aether/hibernate.nix`) hardcodes `boot.resumeDevice` UUID and `resume_offset`. If `/swapfile` is recreated, recompute the offset (`sudo filefrag -v /swapfile | awk 'FNR==4{print $4}' | sed 's/\.\.//'`) before rebuilding, or hibernate-resume will silently fail.
- `secrets/secrets.nix` lists the host SSH public keys allowed to decrypt each `.age` file. After adding a new host or rotating a key, update this file AND re-key (`agenix -r`) — otherwise the host can't read its own secrets at activation time.
- `ikuyo` uses GRUB with `efiSupport = false` (BIOS boot via the GPT EF02 partition declared in `disko.nix`). Don't flip on `efiSupport` without also reshaping the disk layout.

## Project Overview

A Nix flakes-based NixOS configuration for two hosts:
- **`aether`** — AMD laptop with full desktop (Niri Wayland, gaming, IME, hibernate-to-swapfile, home-manager).
- **`ikuyo`** — Hetzner Cloud VPS (qemu-guest, BIOS/GRUB, ext4 via [disko](https://github.com/nix-community/disko), SSH-only login, secrets via [agenix](https://github.com/ryantm/agenix), no home-manager).

The flake is assembled with [`flake-parts`](https://flake.parts) and [`import-tree`](https://github.com/vic/import-tree): every `.nix` file under `./modules/` is auto-discovered and merged into the flake. Each system-level module declares itself as `flake.nixosModules.<name>`; each host's `configuration.nix` then composes the desired modules via `self.nixosModules.<name>`. User environment (packages, dotfiles, shell) is layered on top via home-manager (`home/`), kept out of the auto-import tree and pulled in explicitly from the host config — currently only `aether` wires home-manager. External programs without nixpkgs packages are pulled in via flake inputs (noctalia shell, zen-browser, nix-gaming, emtodo). Remote provisioning uses `nixos-anywhere` + `disko`; secrets are encrypted at rest with `agenix`.

**Stack:** Nix flakes, flake-parts, import-tree, NixOS unstable, home-manager, disko, nixos-anywhere, agenix, Niri (Wayland) + SDDM, Zsh + Nushell + Starship, Zellij, Alacritty/Kitty, Neovim (AstroNvim), Jujutsu + Git/gh, fcitx5 (mozc + Chinese addons), Pipewire, Bluetooth, Steam + Gamescope + Gamemode, auto-cpufreq, hibernate-to-swapfile, OpenSSH (ikuyo).
**Entry point:** `flake.nix` → `flake-parts.lib.mkFlake` → `import-tree ./modules` → `nixosConfigurations.{aether,ikuyo}`.

## Architecture

```
flake.nix
    │   inputs: nixpkgs (unstable), flake-parts, import-tree, home-manager,
    │           quickshell, noctalia, zen-browser, nix-gaming, emtodo,
    │           disko, nixos-anywhere, agenix
    │   outputs = flake-parts.mkFlake (import-tree ./modules)
    │
modules/                                  ← every .nix here is auto-imported by import-tree
    ├── features/                         (reusable NixOS modules; each defines flake.nixosModules.<name>)
    │     ├── niri.nix                    (Niri Wayland + SDDM session)
    │     ├── font.nix                    (system fonts + fontconfig defaults)
    │     ├── fcitx5.nix                  (fcitx5 IME: mozc + Chinese addons)
    │     ├── gaming.nix                  (Steam, Gamescope, Gamemode, 32-bit graphics, auto-cpufreq, dw-proton)
    │     └── optimize.nix                (weekly GC, auto-optimise-store, GRUB limit)
    └── hosts/
          ├── aether/                     (laptop)
          │     ├── default.nix           (flake.nixosConfigurations.aether,
          │     │                          wires home-manager with extraSpecialArgs = { inputs; })
          │     ├── configuration.nix     (flake.nixosModules.aether-config:
          │     │                          imports feature modules + boot/network/users/audio/bluetooth)
          │     ├── hardware.nix          (flake.nixosModules.aether-hardware — AUTO-GENERATED, never edit)
          │     └── hibernate.nix         (swapfile + boot.resumeDevice/resume_offset + lid → hibernate)
          └── ikuyo/                      (Hetzner VPS, headless)
                ├── default.nix           (flake.nixosConfigurations.ikuyo — no home-manager)
                ├── configuration.nix     (flake.nixosModules.ikuyo-config:
                │                          imports hardware + disko + optimize + agenix;
                │                          GRUB BIOS, OpenSSH key-only, sudo-with-password,
                │                          firewall TCP 22 only, dnkyr user with hashedPasswordFile)
                ├── disko.nix             (flake.nixosModules.ikuyo-disko: GPT, EF02 boot + ext4 root on /dev/sda)
                └── hardware.nix          (flake.nixosModules.ikuyo-hardware — AUTO-GENERATED qemu-guest profile)

secrets/                                  (agenix-encrypted secrets, decrypted at activation)
    ├── secrets.nix                       (maps each .age file → list of host SSH pubkeys allowed to decrypt)
    └── ikuyo-dnkyr-password.age          (hashed password for the `dnkyr` user on ikuyo)

home/                                     ← NOT auto-imported; referenced explicitly from the host
    ├── base.nix                          (cross-host home defaults: util packages, mpris-proxy,
    │                                      direnv, git, gh, jujutsu, fastfetch, home.stateVersion)
    ├── hosts/aether/home.nix             (per-host: imports base + cli + gui modules,
    │                                      symlinks configs/{nvim,niri}, .clang-format, emtodo)
    ├── wallpapers/                       (wallpaper images referenced by noctalia)
    └── modules/
          ├── cli/                        (terminal-only tools; safe to import on any host)
          │     ├── default.nix           (imports shell + claude.nix + neovim.nix)
          │     ├── neovim.nix            (neovim + LSPs: nil, lua-ls, pyright, clang-tools, rust-analyzer)
          │     ├── claude.nix            (claude-code, opencode)
          │     └── shell/
          │           ├── default.nix     (imports shell.nix + terminal.nix)
          │           ├── shell.nix       (zsh + oh-my-zsh, nushell, starship, zoxide; CLI tools: bat, eza, yazi…)
          │           └── terminal.nix    (zellij — terminal multiplexer, no GUI deps)
          └── gui/                        (graphical apps; only safe with a display server)
                ├── default.nix           (imports gaming/noctalia/zen-browser/terminal + GUI app pkgs:
                │                          chromium, firefox, tor-browser, discord, telegram-desktop,
                │                          obsidian, zed-editor, foliate, vlc, obs-studio, niri runtime deps)
                ├── terminal.nix          (alacritty + kitty — GUI terminal emulators)
                ├── noctalia.nix          (desktop shell via noctalia flake input + wallpaper config)
                ├── zen-browser.nix       (zen-browser + policies, extensions, search engines)
                └── gaming.nix            (osu-lazer-bin via nix-gaming, heroic, prismlauncher)

configs/                                  (raw dotfiles, symlinked via xdg.configFile in home/hosts/aether/home.nix)
    ├── niri/config.kdl
    └── nvim/                             (AstroNvim lua config)
```

**Key invariants:**
- `modules/` is auto-discovered by `import-tree`. Any `.nix` file dropped under it is imported as a flake-parts module — keep it free of dead code.
- Feature modules under `modules/features/` follow the pattern `{ ... }: { flake.nixosModules.<name> = { ... }: { ...NixOS options... }; };`. To consume one, add `self.nixosModules.<name>` to `imports` in the relevant host's `configuration.nix`.
- Each host gets its own subdir under `modules/hosts/<host>/` with `default.nix` (declares `flake.nixosConfigurations.<host>`) + `configuration.nix` (declares `flake.nixosModules.<host>-config`) + any host-specific helper modules (e.g. `disko.nix`, `hibernate.nix`). Cross-host code lives in `modules/features/`.
- Home-manager is wired per-host in `modules/hosts/<host>/default.nix`; `home/` files are NOT auto-imported. New home modules go under `home/modules/cli/` (terminal-only) or `home/modules/gui/` (needs display server) and are pulled in by name from the host's `home/hosts/<host>/home.nix`. Only `aether` currently uses home-manager — `ikuyo` is system-only.
- `modules/` is for NixOS system-level options only; user-space belongs in `home/modules/`.
- `configs/` is for raw non-Nix config files only; Nix-managed program settings go inline in the `.nix` module.
- `flake.nix` passes `inputs` to the flake-parts tree (outer-module args `{ self, inputs, ... }`) and to home-manager via `extraSpecialArgs = { inherit inputs; }` — home modules destructure `inputs` directly.
- `secrets/secrets.nix` is the access-control list: every `.age` file must list the SSH public keys (from `/etc/ssh/ssh_host_ed25519_key.pub` of each consuming host, plus your editing key) allowed to decrypt it. After editing, run `agenix -r` to re-encrypt with the current key set.

## Decision Tables

### Where to add a new package

| Condition | Where to add | Reason |
|-----------|-------------|--------|
| User-space CLI tool useful on every host | `home/modules/cli/` (new file or existing) | Cross-host, terminal-only, safe even on headless ikuyo if it ever gets home-manager |
| User-space GUI app (needs display server) | `home/modules/gui/` (new file or existing) | Keeps GUI deps out of headless hosts |
| User-space app/utility with no config | `home.packages` in `home/base.nix` (cross-host) or in the appropriate `cli/`/`gui/default.nix` |  |
| User-space app specific to `aether` only | `home.packages` in `home/hosts/aether/home.nix` | Keeps shared modules portable |
| User-space tool with non-trivial config | New `home/modules/{cli,gui}/<name>.nix`, imported via the matching `default.nix` | Keeps host file thin |
| Tool provided by a flake input (not nixpkgs) | New or existing `home/modules/{cli,gui}/<name>.nix` with `inputs` in args | Flake-sourced packages reference `inputs.<name>.packages.${pkgs.stdenv.hostPlatform.system}.<pkg>` (see `home/modules/gui/gaming.nix`) |
| System-level service or hardware config (reusable across hosts) | New `modules/features/<name>.nix` defining `flake.nixosModules.<name>`, then add `self.nixosModules.<name>` to the consuming host's `configuration.nix` imports | flake-parts auto-discovers the file; each host explicitly opts in |
| System-level config truly specific to one host | Inline in `modules/hosts/<host>/configuration.nix`, or a new `modules/hosts/<host>/<topic>.nix` | Host-private, not reusable |
| Font | `modules/features/font.nix` | Fonts are system-level (fontconfig) |
| Encrypted credential / password / API key | New `secrets/<name>.age` (created via `agenix -e`), referenced from a host's `configuration.nix` as `age.secrets.<name>.file = ../../../secrets/<name>.age;` | Plaintext secrets must never enter the Nix store / git history |

### Nix-managed settings vs raw config file in `configs/`

| Condition | Approach | Reason |
|-----------|----------|--------|
| home-manager has a `programs.<name>` module | Use `programs.<name>.settings` / `programs.<name>.enable` | Type-checked, no manual path management |
| Tool has complex config (Lua, KDL) that home-manager can't fully express | Put in `configs/<name>/`, symlink via `xdg.configFile` | Avoids awkward string escaping; config is directly editable |
| Niri window rules / keybindings | `configs/niri/config.kdl` | KDL format not suitable for Nix inline |
| Neovim / AstroNvim lua | `configs/nvim/` | Complex Lua not worth inlining |

### When to create a new flake input vs use nixpkgs

| Condition | Choice | Reason |
|-----------|--------|--------|
| Package exists in nixpkgs unstable | Use nixpkgs | Simpler, benefits from nixpkgs cache |
| Package not in nixpkgs or needs unreleased version | Add flake input with `inputs.nixpkgs.follows = "nixpkgs"` | Ensures consistent nixpkgs version across inputs |
| External project provides a home-manager / NixOS module (noctalia, zen-browser, agenix, disko) | Flake input | The module API requires the flake |

### Adding a new host

| Step | Action |
|------|--------|
| 1. Create `modules/hosts/<host>/default.nix` | Declare `flake.nixosConfigurations.<host> = inputs.nixpkgs.lib.nixosSystem { modules = [ self.nixosModules.<host>-config ... ]; };` |
| 2. Create `modules/hosts/<host>/configuration.nix` | Declare `flake.nixosModules.<host>-config = { ... }: { imports = [ self.nixosModules.<host>-hardware ... ]; ... };` |
| 3. Generate hardware config | On the target machine: `nixos-generate-config --root /mnt --no-filesystems`, then move to `modules/hosts/<host>/hardware.nix` and wrap in the flake-parts pattern (`flake.nixosModules.<host>-hardware = ...`) |
| 4. (Optional) Add disko config | `modules/hosts/<host>/disko.nix` declaring `flake.nixosModules.<host>-disko` (used by nixos-anywhere) |
| 5. (Optional) Wire home-manager | In `default.nix`, add `inputs.home-manager.nixosModules.home-manager` + `home-manager.users.<user> = import ../../../home/hosts/<host>/home.nix;` |
| 6. Add the host's SSH pubkey to `secrets/secrets.nix` | Required if the host needs to decrypt any agenix secret |

## Common Pitfalls

### Forgetting `inputs` in a home-manager module
**Root cause:** Home modules auto-receive `config`, `pkgs`, `lib`, etc., but `inputs` is only available because the host wires `home-manager.extraSpecialArgs = { inherit inputs; }` in `modules/hosts/aether/default.nix`. You still have to destructure it.
**Symptom:** `error: attribute 'inputs' missing` at eval time.
**Fix:** Add `inputs` to the module's function args: `{ pkgs, inputs, ... }:`. See `home/modules/gui/gaming.nix`, `noctalia.nix`, `zen-browser.nix`, and `home/hosts/aether/home.nix` for examples.

### Wrong outer args in a feature module (flake-parts vs NixOS)
**Root cause:** Feature modules under `modules/features/` and host modules under `modules/hosts/<host>/` are flake-parts modules, not NixOS modules. The outer function receives flake-parts args (`{ self, inputs, ... }`) and must declare `flake.nixosModules.<name> = { ...NixOS args... }: { ... };`. Trying to put NixOS options at the top level (or destructuring `pkgs` in the outer args) won't work.
**Symptom:** `error: attribute 'pkgs' missing` in the outer function, or options like `services.X` being silently ignored.
**Fix:** Match the pattern in existing files (`modules/features/niri.nix`, `optimize.nix`, `modules/hosts/ikuyo/disko.nix`): outer `{ ... }:` returns `{ flake.nixosModules.<name> = { config, pkgs, ... }: { ... }; }`. Then add `self.nixosModules.<name>` to the consuming host's imports.

### Adding a feature module but forgetting to import it
**Root cause:** `import-tree` discovering a file under `modules/` only registers it (e.g. as `flake.nixosModules.foo`). No host actually uses it until you list `self.nixosModules.foo` in the `imports` of the relevant host's `configuration.nix` (`aether-config` or `ikuyo-config`).
**Symptom:** New module's options have no effect after rebuild; no eval error.
**Fix:** Add the entry to the consuming host's `imports = [ ... ]` list.

### Putting a GUI-dependent home module under `home/modules/cli/`
**Root cause:** The `cli/` vs `gui/` split exists so headless hosts can import only the cli set without pulling in a display server. A module that quietly depends on Wayland/X11 (e.g. references `pkgs.alacritty`, sets `wayland.windowManager.*`) breaks that contract.
**Symptom:** Closure size balloons on a headless host, or `nix build` pulls in graphics stacks unexpectedly.
**Fix:** Move the offender to `home/modules/gui/` and import it from `gui/default.nix`. Reserve `cli/` for tools that work in a TTY.

### Editing deployed dotfiles instead of the source
**Root cause:** `~/.config/niri/config.kdl` and `~/.config/nvim/` are symlinks managed by home-manager pointing into the Nix store (sourced from `configs/`, wired via `xdg.configFile."<name>".source` in `home/hosts/aether/home.nix`). Direct edits are either ignored or lost on next switch.
**Symptom:** Changes don't persist after `nixos-rebuild switch`.
**Fix:** Edit the source files in `configs/niri/` or `configs/nvim/`, then rebuild.

### Wallpaper path relies on the /etc/nixos symlink
**Root cause:** `home/modules/gui/noctalia.nix` hardcodes the wallpaper path as `/etc/nixos/home/wallpapers/Anime-Girl-Rain.png`. This works because `/etc/nixos` is a symlink to `/home/dnkyr/nixos-config`. Wallpaper images live in `home/wallpapers/`.
**Symptom:** Wallpaper not found if the repo is ever moved or the symlink breaks.
**Fix:** If the repo moves, update the path in `home/modules/gui/noctalia.nix` and recreate the `/etc/nixos` symlink.

### `useOSProber = true` requires other OS to be mounted at boot
**Root cause:** GRUB's OS prober scans for other bootable partitions. If Windows or another OS partition isn't visible during build, it may not appear in GRUB.
**Symptom:** Dual-boot entry disappears from GRUB menu.
**Fix:** This is expected behavior; rebuild while the partition is accessible.

### agenix secret unreadable on a host after key rotation or new host
**Root cause:** Each `.age` file is encrypted to the exact set of public keys in `secrets/secrets.nix` at the time of `agenix -e`. If a host's SSH host key changes (reinstall, fresh deploy), or a new host is added, existing secrets stay encrypted to the old key set.
**Symptom:** Activation fails on the host with `agenix: failed to decrypt /run/agenix/<name>` or the user has no usable password.
**Fix:** Update `secrets/secrets.nix` with the current host pubkey (`cat /etc/ssh/ssh_host_ed25519_key.pub` on the host), then `cd secrets && agenix -r` to re-encrypt every secret to the new key set. Commit the updated `.age` files.

### nixos-anywhere wipes the disk per `disko.nix`
**Root cause:** `nixos-anywhere --flake .#ikuyo` runs the `ikuyo-disko` module, which formats `/dev/sda` end-to-end (GPT, EF02 boot, ext4 root). This is the intended bootstrap flow for a fresh VPS but is destructive on any existing data.
**Symptom:** All data on the target host is gone after a single command.
**Fix:** Only run `nixos-anywhere` against a host you actually intend to wipe. For routine updates of an already-installed `ikuyo`, use `nixos-rebuild switch --flake .#ikuyo --target-host root@<ip>` instead — that respects the existing filesystem.
